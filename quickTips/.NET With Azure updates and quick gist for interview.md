# ✅ .NET Interview Questions (Latest)

## 1️⃣ What’s new in .NET 10 and why is it important?

**Answer:**

*   .NET 10 is an **LTS release** meant for long‑term enterprise support.
*   Focuses on **performance (JIT, NativeAOT)**, **AI integration**, **cloud‑native tooling**, and **developer productivity**.
*   Aligns ASP.NET Core, EF Core, MAUI, and Aspire for cloud readiness.

✅ *Why interviewers care:* production stability + performance.

***

## 2️⃣ What improvements came with C# 14?

**Answer:**

*   **Extension members** (properties/operators via extensions).
*   Partial constructors/events → better **source generator support**.
*   Better `Span<T>` performance and reduced boilerplate.
*   More expressive and efficient syntax.

✅ *Talk point:* “Cleaner, faster, and more extensible C#.”

***

## 3️⃣ What is NativeAOT and when should you use it?

**Answer:**

*   Compiles .NET apps **ahead‑of‑time** to native binaries.
*   Benefits: **faster startup, lower memory, smaller containers**.
*   Ideal for **microservices, serverless, and CLI tools**.

⚠️ Trade‑off: limited reflection/dynamic features.

***

## 4️⃣ How does ASP.NET Core support cloud‑native apps?

**Answer:**

*   Minimal APIs, built‑in health checks, OpenTelemetry.
*   First‑class container & Kubernetes support.
*   Works seamlessly with **Azure App Service, AKS, and Azure Functions**.

***

## 5️⃣ What is .NET Aspire?

**Answer:**

*   A framework for building **distributed cloud‑native systems**.
*   Handles service discovery, observability, config, resilience.
*   Designed for microservices running on Azure.

✅ *Say this:* “Aspire simplifies microservices without heavy Kubernetes plumbing.”

## ✅ Scenario‑Based Questions (Very Important)

## 11️⃣ How would you deploy a .NET microservice on Azure?

**Ideal Answer:**

1.  Build ASP.NET Core service (.NET 10).
2.  Containerize using Docker.
3.  Deploy to **AKS** or **Azure App Service**.
4.  Use Azure Monitor + Application Insights.
5.  Secure with Entra ID & Managed Identity.

***

## 12️⃣ How do you handle performance optimization in .NET?

**Answer:**

*   Reduce allocations (Span, pooling).
*   Async/await properly.
*   Use profiling (dotnet-trace, PerfView).
*   Enable NativeAOT where applicable.

***

## 13️⃣ How do you secure secrets in Azure?

**Answer:**

*   Use **Azure Key Vault**.
*   Access via **Managed Identity** (no secrets in code).
*   Entra ID for authentication.
